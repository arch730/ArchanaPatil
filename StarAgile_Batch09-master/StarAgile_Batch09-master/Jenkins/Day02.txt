General - This Job is Parameterized

What is a parameter?
Parameters are used to pass the variables

Types of parameters;
	1. String
	2. Choice

String Parameter - string is a group of characters
Using string parameter we can call variables and print the required output
parameters will be defined as KEY-VALUE pairs

What is a Key-Value pair?
Name = Kastro
Role = DevOps Engineer
Age = 17

Triggers - Build Periodically
---------------------------------------------------------------------------------------------
Triggers are used to schedule the job execution
In this, the job will be built based on the time period we will configure in Jenkins

Everyday - @ 10.00 AM (Mon-Fri), 06 Nov to 16 Nov
To trigger the build we will use CRON Expression

* * * * *	-----> Every minute the job will get executed
H * * * * ----> Every hour the job will get executed
H/2 * * * * ----> Every half an hour the job will get executed


Version 1 code	----> Build and Package
Version 2 code	----> Build and Package

Build Periodically doesn't bother about the code changes in the repo. 
It will continuously build the code and consumes the underlying storage.

---------------------------------------------------------------------------------------------
Triggers - Poll SCM
---------------------------------------------------------------------------------------------
Here also we will cron expression

Poll SCM will look after the changes made in the GitHub repo, and then only it will trigger the job

The limitation with Poll SCM is we need to wait for a minimum of 1 minute for the job to trigger
